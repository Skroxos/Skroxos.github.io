Overview

Drone Assembly Simulator is a Unity project where the player builds a drone by selecting and placing individual components. The simulation validates each assembly step — checking compatibility, correct placement, and completeness before allowing progression.

The entire project was built solo: from initial architecture decisions through to gameplay, UI, and final polish. No templates, no teammates — full ownership of every line.

What I Built

  • Component validation system — runtime checks for compatibility and correct assembly order
  • Modular drone part system — each component defined by data, not hardcoded logic
  • UI feedback layer — communicates validation state clearly without cluttering the scene
  • State management — clean separation between assembly phases using a structured FSM
  • Scene architecture — designed for maintainability and easy extension with new parts

Challenges

The hardest part was designing the validation system to be data-driven rather than case-by-case. Early versions had explicit checks per component type — which broke immediately when adding new parts. Refactoring to a rule-based validator made the system dramatically more scalable and easier to test.

Working solo also meant making all architectural decisions without external review — which forced a more deliberate and documented approach to every design choice.
