Unity 3D C# VR / OpenXR Optimization

The Problem (Výzva)

U VR titulů je největším nepřítelem nestabilní framerate, který způsobuje nevolnost. Hlavním cílem bylo vytvořit plynulý zážitek s nulovými záseky (stuttering) způsobenými Garbage Collectorem.

The Solution (Řešení)

Implementoval jsem architekturu, která minimalizuje alokace na Heapu během hraní. Veškeré vyhledávání objektů probíhá přes Dictionary s O(1) složitostí. Logika herních systémů je striktně oddělena od inputů brýlí přes Eventy, což zajišťuje modularitu.

Ukázka kódu

Zde je příklad, jak řeším [např. Interakci objektů]:


// Příklad tvého čistého kódu
public class InteractionSystem : MonoBehaviour 
{
    // Použití Dictionary pro rychlý přístup bez GC alokací
    private readonly Dictionary _interactables = new();

    public void Register(int id, IInteractable item) => _interactables[id] = item;
}
