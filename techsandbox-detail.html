What Is TechSandbox?

TechSandbox is not a game — it's a controlled environment where I prototype Unity systems and patterns in isolation before they go anywhere near a real project. Think of it as an ongoing sketchbook for architecture decisions, shader ideas, and framework experiments.

Anything that survives here and proves itself gets extracted into actual production code. Anything that doesn't? Still a useful lesson.

Current Focus: Async REST API Framework

The most developed module right now is a production-ready async networking layer replacing the old Coroutine-based approach:

  • All network calls built on UniTask — non-blocking, zero-alloc async/await for Unity
  • CancellationToken propagated through every request for clean cancellation on scene unload or timeout
  • Dependency injection via VContainer — services are registered and resolved without singletons
  • Typed response models with centralized error handling — no scattered null checks
  • Designed to be dropped into any Unity project as a self-contained module

Why It Matters

Having a dedicated sandbox means production code stays clean. Instead of experimenting directly in a live project (and creating tech debt), I validate ideas here first — then bring only proven patterns across. It's also an honest signal of how I think about engineering: deliberate, iterative, and documented.
