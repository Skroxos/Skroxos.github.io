C# Architecture & Network Sandbox

VContainer UniTask REST API Dependency Injection

Mobile games heavily rely on stable backend communication, but standard Unity WebRequests using Coroutines often lead to callback hell, block the main thread, and lack proper timeout fail-safes. This project serves as a production-ready framework to solve these structural issues.

The Challenge

The primary goal was to create a network layer that is completely isolated from the game's UI and core logic. If the backend server is offline or experiencing high latency, the game must not freeze, and the UI must react gracefully. Furthermore, the architecture needed to support rapid offline development without requiring a live server connection.

The Solution

I developed a robust network layer using UniTask for zero-allocation asynchronous programming. To prevent hard-coupling, the UI components communicate with the network layer strictly through an INetworkService interface.

The dependencies are resolved and injected using VContainer. This Dependency Injection setup allows seamless switching between a live production API and a local "Mock" server simulator by simply changing one line in the Lifetime Scope.

Implementation Details

By enforcing the Dependency Inversion Principle (SOLID), the client becomes "dumb". It doesn't know how the data is sent; it only knows the contract it signed. Below is the core interface defining the strict contract for any network service used in the project.

INetworkService.cs
using System.Threading;
using Cysharp.Threading.Tasks;

public interface INetworkService
{
    UniTask<string> SendPostRequestSafeAsync(
        string url, 
        string jsonPayload, 
        CancellationToken token
    );
}
